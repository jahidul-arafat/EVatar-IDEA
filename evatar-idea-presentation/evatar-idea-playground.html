EVAtAR-IDEA treats the placement of expression tags as a combinatorial optimisation problem. An Interactive Distributed Evolutionary Algorithm (IDEA) — grounded in Dozier (2005) — searches a vocabulary of 21 confirmed tokens across position, type, and duration to maximise: F(p) = 0.5·f1 + 0.3·f2 + 0.2·f3 where f1 = AffectNet class accuracy, f2 = timing alignment, f3 = expression consistency.

Three baselines are compared: p_auto (HeyGen Auto-enhance — no fitness function), p_manual (expert human placement), and p* (EVAtAR-IDEA optimised).

Pre-registered threshold: F(p*) ≥ 0.75 · Wilcoxon p < 0.05 · Cohen d ≥ 0.5

P1 CORPUS DESIGN
Methodology, Rationale & References
The corpus uses exactly 8 delivery categories because they map one-to-one to the AffectNet 8-class facial expression taxonomy — the ground truth model used by DeepFace to compute the f1 accuracy score in EVAtAR-IDEA's fitness function F(p). If a category had no corresponding AffectNet class, its sentences would produce unscoreable clips — making the baseline measurement meaningless.
CATEGORY → AFFECTNET CLASS → DEEPFACE SCOREABLE
PRIMARY REFERENCE

AffectNet: A database for facial expression, valence, and arousal computing in the wild.
IEEE Transactions on Affective Computing, 10(1), 18–31.
DOI: 10.1109/TAFFC.2017.2740923
DEEPFACE IMPLEMENTATION
Serengil, S. I. & Ozpinar, A. (2020).
LightFace: A hybrid deep face recognition framework.
ASYU 2020, IEEE.
DeepFace uses AffectNet weights for emotion classification
Known limitation acknowledged in Paper 1: [curious] is not an AffectNet category. AffectNet's 8 classes are: Neutral, Happy, Sad, Surprise, Fear, Anger, Disgust, Contempt (Mollahosseini et al. 2019, Table 3). EVAtAR-IDEA maps [curious] to the Fear class as the closest VAD proxy (high arousal, low-to-negative valence in Russell's circumplex) — this is a researcher inference, not an AffectNet definition. This decision is documented as a threat-to-validity in Paper 1 and pre-registered at OSF. The GA compensates by pairing [curious] with [head:tilt] + [eyebrow:raise] gesture tokens.
Sentence difficulty is assigned using a three-factor rubric combining syntactic complexity (Flesch-Kincaid), emotional ambiguity (VAD model), and expression timing risk (F3 late onset probability). Each factor is independently grounded in published research.
DIFFICULTY ASSIGNMENT FLOWCHART
1
Count Words (W)
Strip punctuation · count tokens · record W
Example: "Today we will explore calculus." → W = 5
2
Compute Flesch-Kincaid Grade (FK)
FK = 0.39×(W/sentences) + 11.8×(syllables/W) − 15.59
Proxy for syntactic complexity and reading demand.
Example: "The meme space stores individuals…" → FK ≈ 17 → Hard
Flesch, R. (1948). A new readability yardstick. Journal of Applied Psychology, 32(3), 221–233.
3
Score VAD (Valence-Arousal-Dominance)
Using NRC Emotion Lexicon · each content word scored on V/A/D scale (0–1). Low dominance + high arousal = emotionally ambiguous = harder for avatar to render consistently.
Example: "It is okay to feel uncertain" → Valence=0.42 (mixed) · Arousal=0.61 (moderate) · Dominance=0.28 (low) → ambiguous → Medium/Hard
Russell, J. A. & Mehrabian, A. (1977). Evidence for a three-factor theory of emotions. Journal of Research in Personality, 11(3), 273–294.
Mohammad, S. & Turney, P. (2013). Crowdsourcing a word-emotion association lexicon. Computational Intelligence, 29(3), 436–465.
4
Estimate F3 Late Onset Risk
Locate primary emotion cue word position P_cue. If P_cue / W > 0.60 → F3 late onset risk flag. Late cues force the avatar to express emotion after the audio has peaked — degrading f2 timing.
Example: "In twelve months… it is finally done." → emotion cue "done" at word 16/19 = 84% → F3 late-onset risk → Hard

A common question: "If real Aubria scripts are 500-word paragraphs, why does the playground use single sentences?" The answer is architectural — HeyGen's render pipeline and DeepFace's scoring model both operate at the sentence-clip level, not the paragraph level.

✂️
Segmented into ~35 sentence-clips
Each sentence becomes an independent render unit. HeyGen processes one sentence at a time — not the whole paragraph. This is how the HeyGen API works: one text input → one video output.
↓ one render job per clip
🎬
HeyGen renders each clip independently
Each clip: one API call, one set of expression tags, ~30–90 seconds render time, token credits consumed. Tags placed in clip N have NO effect on clip N+1 — they are independent.
35 clips × ~60s = ~35 minutes render time. 35 clips × ~12 credits = ~420 credits per script pass.
↓ DeepFace scores each clip
🔬
DeepFace scores each clip independently
DeepFace analyses the video frame-by-frame. It outputs per-frame AffectNet class probabilities for each clip. F(p) is computed per clip — not across the whole paragraph.
F(clip_1) = 0.71, F(clip_2) = 0.63, F(clip_3) = 0.58… Each score is independent.
↓ EVAtAR-IDEA optimises each clip
🧬
EVAtAR-IDEA runs one GA per clip
The sentence IS the optimisation unit. The GA runs independently for each clip, finding the best tag sequence p* for that specific sentence. The meme space allows solutions from one clip to seed clips of the same emotion class — reducing render cost over time.
Clip 1 (neutral) → GA runs 64 renders → p*₁. Clip 2 (neutral, seeds from clip 1) → GA runs ~20 renders → p*₂. Total cost decreases with each same-emotion clip.

Implication for Paper 1 scope: The P1 corpus of 50 sentences represents 50 independent optimisation problems — not 50 sentences from one paragraph. This is intentional: variety across emotion classes and difficulty levels provides statistical power for the Wilcoxon signed-rank test comparing F(p*) vs F(p_manual) across the full distribution, not just one script context. ⌨ ALGORITHM — EVAtAR-IDEA PSEUDOCODE 
ALGORITHM EVAtAR-IDEA(script_corpus S, vocab V, pop_size=8, max_gen=8)

// ── P1: Baseline Measurement ────────────────────────────────
FOR EACH sentence s IN S:
    p_manual ← HumanLabelTags(s, V)           // Stage 2: researcher places tags
    clip     ← HeyGen.Render(s, p_manual)     // HeyGen API call
    F_manual ← DeepFace.Score(clip, s.emotion) // f1·f2·f3 computed
    D1.append(s, p_manual, F_manual)           // build dataset
PRE-REGISTER threshold: F(p*)≥0.75, Wilcoxon p<0.05, Cohen d≥0.5

// ── P2: GA Optimisation ─────────────────────────────────────
FOR EACH sentence s IN S:
    pop ← InitPopulation(pop_size, V)         // 8 random candidate sequences
    IF MemeSpace[s.emotion].exists():
        pop[0:3] ← MemeSpace[s.emotion].top3  // seed from stored solutions

    FOR gen = 1 TO max_gen:
        FOR EACH candidate p IN pop:
            clip    ← HeyGen.Render(s, p)      // render via API
            F(p)    ← DeepFace.Score(clip)     // AffectNet 8-class scoring
            IF F(p) ≥ 0.70:
                MemeSpace[s.emotion].store(p, F(p))
        survivors ← Tournament(pop, k=3)       // select best candidates
        pop       ← Mutate(survivors, V)       // shift_pos · change_tag · add · remove · change_dur
        pop       ← ConstraintRepair(pop)      // fix F2 stacking, F3 late onset

    p* ← argmax F(p) IN pop                  // best candidate sequence
    D2.append(s, p*, F(p*), F_manual, delta)

COMPARE F(p*) vs F(p_manual) via Wilcoxon signed-rank + Cohen d

// ── P3: Cross-Platform ──────────────────────────────────────
FOR EACH sentence s, optimised p* IN D2:
    F_heygen ← DeepFace.Score(HeyGen.Render(s, p*))
    F_did    ← DeepFace.Score(DID.Render(s, p*))    // NL translation of tags
    F_akool  ← DeepFace.Score(Akool.Render(s, p*))  // audio-driven only
    D3.append(s, F_heygen, F_did, F_akool)
REPORT cross-platform degradation · T2 failure differences
📋 F1–F8 FAILURE TAXONOMY
Eight documented failure modes for scripted avatar expression tags. Pre-registered as part of Paper 1 (D1 dataset). F(p) = 0.5·f1 + 0.3·f2 + 0.2·f3 is independent of F1–F8; the taxonomy informs constraint repair in the GA.
