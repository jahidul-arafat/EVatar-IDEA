EVAtAR
The Problem
Scripted avatar videos
look flat, robotic,
and emotionally absent.
During production of 4–10 minute Aubria lecture videos, every sentence required manual expression tag placement. Each attempt broke something — a frozen face, a misaligned expression peak, an emotion that fired too late or clashed with adjacent tags.

The result: 200+ hours of manual iteration across 100+ re-renders to produce a single polished video. Every re-render consumed HeyGen credits — making the process not just time-consuming but financially costly at scale.

The root cause: no systematic method exists for deciding which expression tags to place, where in the sentence, and for how long — leading to eight documented failure modes (F1–F8) that consistently degrade the avatar's facial expression quality.

Research Objective
Optimise expression tag
sequences automatically
using evolutionary search.
EVAtAR-IDEA treats the placement of expression tags as a combinatorial optimisation problem. An Interactive Distributed Evolutionary Algorithm (IDEA) searches a vocabulary of 21 confirmed tokens across position, type, and duration dimensions to maximise a composite perceptual fitness score F(p) = 0.5·f1 + 0.3·f2 + 0.2·f3 — where f1 is AffectNet accuracy, f2 is timing alignment, and f3 is expression consistency.

Pre-registered threshold: F(p*) ≥ 0.75 · Wilcoxon p < 0.05 · Cohen d ≥ 0.5

ALGORITHM EVAtAR-IDEA(script_corpus S, vocab V, pop_size=8, max_gen=8)

// ── P1: Baseline Measurement ────────────────────────────────
FOR EACH sentence s IN S:
    p_manual ← HumanLabelTags(s, V)           // Stage 2: researcher places tags
    clip     ← HeyGen.Render(s, p_manual)     // HeyGen API call
    F_manual ← DeepFace.Score(clip, s.emotion) // f1·f2·f3 computed
    D1.append(s, p_manual, F_manual)           // build dataset
PRE-REGISTER threshold: F(p*)≥0.75, Wilcoxon p<0.05, Cohen d≥0.5

// ── P2: GA Optimisation ─────────────────────────────────────
FOR EACH sentence s IN S:
    pop ← InitPopulation(pop_size, V)         // 8 random candidate sequences
    IF MemeSpace[s.emotion].exists():
        pop[0:3] ← MemeSpace[s.emotion].top3  // seed from stored solutions

    FOR gen = 1 TO max_gen:
        FOR EACH candidate p IN pop:
            clip    ← HeyGen.Render(s, p)      // render via API
            F(p)    ← DeepFace.Score(clip)     // AffectNet 8-class scoring
            IF F(p) ≥ 0.70:
                MemeSpace[s.emotion].store(p, F(p))
        survivors ← Tournament(pop, k=3)       // select best candidates
        pop       ← Mutate(survivors, V)       // shift_pos · change_tag · add · remove · change_dur
        pop       ← ConstraintRepair(pop)      // fix F2 stacking, F3 late onset

    p* ← argmax F(p) IN pop                  // best candidate sequence
    D2.append(s, p*, F(p*), F_manual, delta)

COMPARE F(p*) vs F(p_manual) via Wilcoxon signed-rank + Cohen d

// ── P3: Cross-Platform ──────────────────────────────────────
FOR EACH sentence s, optimised p* IN D2:
    F_heygen ← DeepFace.Score(HeyGen.Render(s, p*))
    F_did    ← DeepFace.Score(DID.Render(s, p*))    // NL translation of tags
    F_akool  ← DeepFace.Score(Akool.Render(s, p*))  // audio-driven only
    D3.append(s, F_heygen, F_did, F_akool)
REPORT cross-platform degradation · T2 failure differences
P1 — BASELINE MEASUREMENT

For each of 50 sentences, a human researcher (Jahidul) places expression tags word-by-word. HeyGen renders the clip. DeepFace analyses it frame-by-frame using the AffectNet 8-class model. The score F(p_manual) = 0.5·f1 + 0.3·f2 + 0.2·f3 is recorded. This builds the D1 dataset and establishes what an expert human can achieve — the baseline the GA must beat. Eight failure modes (F1–F8) are documented and used to define the constraint repair handler.

→ Paper 1 · Vocabulary & Corpus Study
P2 — GA OPTIMISATION (EVAtAR-IDEA)

For each sentence, EVAtAR-IDEA creates 8 candidate tag sequences from the 21-token vocabulary. Each candidate is rendered by HeyGen and scored by DeepFace — exactly the same pipeline as P1. The best-scoring candidates survive, get mutated (position shifted, tag swapped, timing changed), and compete again. After 8 generations, the winner p* is compared to p_manual. Solutions above 0.70 are stored in the Meme Space and reused as starting points for sentences of the same emotion — so the system gets faster over time.

→ Paper 2 · Optimisation Study · Pre-reg: F(p*)≥0.75 · Wilcoxon · Cohen d
P3 — CROSS-PLATFORM PORTABILITY

The optimised p* sequences from P2 are tested on three avatar platforms. HeyGen ★★★ supports inline expression tags with precise control — best F(p) scores. D-ID ★★ uses natural-language motion hints instead of tags — expected ~15–25% score degradation. Akool ★ is audio-driven only with no expression control — lower bound on F(p). The same p* sequence and the same DeepFace scoring pipeline are used for all three. This shows whether EVAtAR-IDEA's optimisation is platform-specific or generalisable.

GA Terminology note: In Genetic Algorithms, an "individual" means a candidate solution — one possible tag sequence being tested. It has nothing to do with people or research participants. A "population" is the set of 8 candidate sequences competing in one generation. The GA evolves better candidates over 8 generations by keeping the best-scoring ones and mutating them.
